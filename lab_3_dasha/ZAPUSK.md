# Как запустить проект для защиты лабораторной работы

Рабочее приложение с заданиями (параметры URL, заголовки, cookie, форма, проверка телефона) находится в папке **`app`**. Папка **`SDO_template`** — это только шаблон с курса; для демонстрации используйте проект из **`app`**.

## Что установить один раз

Нужны **Python 3** и пакеты из **`requirements.txt`** (Flask, Faker, pytest для наглядного вывода тестов). Из **корня** проекта:

```powershell
cd "C:\Users\dasha\VUZ\4_Semestr_2026\Web\lab_2_dasha"
python -m pip install -r requirements.txt
```

(Путь замените на свой, если папка лежит в другом месте.)

## Запуск сайта в браузере

### Вариант 1 — двойной щелчок по `run.bat` в корне проекта

Скрипт сам перейдёт в папку `app` и запустит сервер.

### Вариант 2 — команды вручную

```powershell
cd "C:\Users\dasha\VUZ\4_Semestr_2026\Web\lab_2_dasha\app"
$env:FLASK_APP = "app"
python -m flask run
```

В **cmd.exe** вместо `$env:FLASK_APP = "app"` используйте:

```cmd
set FLASK_APP=app
python -m flask run
```

Откройте в браузере: **http://127.0.0.1:5000/**

Полезные страницы для показа по заданию:

- главная — список ссылок;
- **/params** — параметры URL (можно добавить `?a=1&b=2`);
- **/headers** — заголовки запроса;
- **/cookie** — установка/удаление демо-куки;
- **/form** — форма и вывод отправленных полей;
- **/check_phone** — проверка и форматирование номера телефона.

Чтобы остановить сервер, в том же окне терминала нажмите **Ctrl+C**.

## Запуск автотестов (п. 3 задания)

Из **корня** проекта (где лежат `task.md` и папка `tests`).

### С подробным выводом по каждому тесту (удобно для защиты)

Нужен установленный **pytest** (он есть в `requirements.txt`). В терминале будет видно, как проверяется каждый тест, а в конце — сводка, например **`17 passed`** («столько-то из стольких пройдено»):

```powershell
cd "C:\Users\dasha\VUZ\4_Semestr_2026\Web\lab_2_dasha"
python -m pytest tests/ -v
```

Флаг **`-v`** (*verbose*) как раз включает построчный вывод имён тестов и статуса `PASSED` / `FAILED`.

### Без pytest (только стандартная библиотека)

```powershell
cd "C:\Users\dasha\VUZ\4_Semestr_2026\Web\lab_2_dasha"
python -m unittest discover -s tests -p "test_*.py" -v
```

В конце будет строка вида `Ran 17 tests` и `OK` — тоже понятно, но итог в стиле «N passed» даёт именно **pytest**.

---

**Кратко для устного ответа:** «Ставлю зависимости из `requirements.txt`, захожу в каталог `app`, задаю `FLASK_APP=app`, запускаю `python -m flask run` и открываю http://127.0.0.1:5000 в браузере. Тесты — `python -m pytest tests/ -v` из корня репозитория.»

**Про тесты — что сказать на защите.** После команды `pytest` с флагом `-v` в терминале по очереди появляются строки: имя каждого теста и рядом **`PASSED`** — значит проверка прошла. Справа в квадратных скобках растёт процент (`[100%]` в конце). В самом низу одна итоговая строка, например **`17 passed`** — это число успешно пройденных тестов из собранных; если что-то упало, будет **`failed`** и указание, какой тест не прошёл.

**Что важно проговорить по смыслу:** тесты написаны по пункту 3 задания — проверяют страницу параметров URL (все query-параметры на экране), страницу заголовков (имя и значение, в том числе свой заголовок и `Host`), страницу cookie (сценарий «куки нет — ставим» и «куки есть — снимаем» через два запроса подряд), форму (после POST в ответе видны введённые поля), а также проверку телефона: допустимые форматы из примера в задании, две разные ошибки по тексту (цифры и недопустимые символы), классы Bootstrap **`is-invalid`** и **`invalid-feedback`** при ошибке и вывод отформатированного номера при успехе. В задании требовалось не менее 15 тестов — у нас их больше, все они в файле `tests/test_lab2.py`.
